# Repo-rter v0.4.4 — OS 키체인 보안 저장소 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** GitHub PAT, WebDAV 패스워드, E2EE 패스프레이즈를 웹뷰 `localStorage` 평문 저장에서 OS 네이티브 키체인(macOS Keychain / Windows Credential Manager / Linux Secret Service)으로 옮기고, 기존 사용자의 평문 시크릿을 안전하게 이관한다.

**Architecture:** Rust `keyring` 크레이트를 직접 감싼 Tauri 커맨드 3개(`set_secret`/`get_secret`/`delete_secret`)를 만들고, TS `src/lib/secrets.ts`가 유일한 게이트웨이가 된다. Tauri 안에서는 키체인, 브라우저에서는 기존 localStorage로 분기한다. 읽기 경로가 키체인 미스 시 레거시 평문을 자동으로 읽어주므로 마이그레이션 완료 여부와 무관하게 앱이 동작한다 — 이 설계가 부팅 시 마이그레이션과 화면 렌더 사이의 경쟁 조건을 원천 제거한다.

**Tech Stack:** Rust `keyring` 4.1.5 / Tauri 2.11.5 / TypeScript 5 / React 19 / vitest 4

## Global Constraints

프로젝트 전역 요구사항. 모든 태스크의 요구사항에 암묵적으로 포함된다.

- **의존성은 아래 값을 정확히 사용한다** (2026-07-17 crates.io 실측, 추측 금지):
  - `keyring = "4.1.5"` — **기본 feature를 그대로 쓴다.** `default = ["v1"]`이고 `v1`이 플랫폼 백엔드(apple-native / windows-native / zbus-secret-service)를 이미 포함한다. `default-features = false`를 쓰면 백엔드가 사라진다.
- **`keyring::v1` API는 아래가 전부다** (실제 컴파일·실행으로 검증됨):
  - `Entry::new(service: &str, username: &str) -> Result<Entry>`
  - `entry.set_password(&str) -> Result<()>` / `entry.get_password() -> Result<String>` / `entry.delete_credential() -> Result<()>`
  - `delete_password`라는 메서드는 **없다.** 없는 항목은 `Error::NoEntry`로 온다.
  - `v1`은 플랫폼 store를 자동 선택한다. 초기화 호출이 필요 없다.
- **`@tauri-apps/plugin-secure-storage`를 쓰지 않는다.** npm에 존재하지 않는다(404). JS 바인딩 없이 `invoke()`로 직접 호출한다.
- **Linux 빌드에 새 시스템 패키지가 필요 없다.** `cargo tree --target x86_64-unknown-linux-gnu`로 확인한 백엔드 체인은 `zbus-secret-service-keyring-store → secret-service 5.1 → zbus 5.17`로 전부 순수 Rust다. C 라이브러리 링크가 없다. **`.github/workflows/release.yml`의 apt 목록을 건드리지 말 것.** (v0.4.3 기획안에 "libsecret-1-dev가 필요하다"고 적혀 있으나 그건 keyring 3.x 얘기이고 **틀렸다.** Task 6에서 그 문서를 바로잡는다.)
- **시크릿 3종만 이관한다:** `github_pat`, `sync_webdav_pass`, `sync_encryption_key`. `sync_webdav_url`, `sync_webdav_user`, `sync_provider`, `data_retention_days` 등은 시크릿이 아니므로 localStorage에 그대로 둔다.
- **키체인 쓰기 실패 시 localStorage로 폴백하지 않는다.** 조용히 평문으로 쓰면 이 릴리스의 목적이 사라진다. Tauri 안에서는 에러를 그대로 올린다. (읽기 경로의 레거시 폴백은 예외 — 아래 참조)
- **브라우저(비-Tauri) 경로는 localStorage를 그대로 쓴다.** 이 앱은 `output: 'export'`라 `npm run dev`로 브라우저에서도 돈다. 브라우저에는 키체인이 없다. 실제 배포물은 항상 Tauri이므로 사용자 노출은 없다.
- **`sync_encryption_key`를 잃으면 원격 백업을 영구히 복호화할 수 없다.** 마이그레이션은 반드시 `키체인 쓰기 → 읽어서 대조 → 일치할 때만 평문 삭제` 순서를 지킨다.
- 커밋 메시지는 Conventional Commits를 따른다.

### Task 1: Rust 키체인 커맨드

`tauri-plugin-secure-storage`(커뮤니티 크레이트, 최근 다운로드 1,437회, crates.io상 저장소 URL 없음)를 쓰지 않고 그것이 감싸는 `keyring`을 직접 쓴다. `keyring`은 최근 다운로드 650만 회에 실제 저장소(`open-source-cooperative/keyring-rs`)가 있는 주류 크레이트다. 커맨드 3개면 끝나므로 중간 계층을 둘 이유가 없다.

`lib.rs`는 이미 320줄이므로 새 모듈로 분리한다.

**Files:**
- Create: `src-tauri/src/secrets.rs`
- Modify: `src-tauri/src/lib.rs` (맨 위 `mod secrets;`, `invoke_handler` 목록)
- Modify: `src-tauri/Cargo.toml` (`[dependencies]`)

**Interfaces:**
- Consumes: 없음
- Produces: Tauri 커맨드 3개. Task 2의 `src/lib/secrets.ts`가 유일한 소비자다.
  - `set_secret(key: String, value: String) -> Result<(), String>`
  - `get_secret(key: String) -> Result<Option<String>, String>` — 없으면 `Ok(None)`
  - `delete_secret(key: String) -> Result<(), String>` — 없어도 `Ok(())`
  - JS에서 `invoke('set_secret', { key, value })`처럼 camelCase 인자로 호출한다.

- [ ] **Step 2: 커맨드 모듈 작성**

`src-tauri/src/secrets.rs` 생성.

```rust
use keyring::v1::{Entry, Error};

/// 키체인 항목의 service 이름. tauri.conf.json의 identifier와 맞춘다.
const SERVICE: &str = "com.reporter.app";

#[tauri::command]
pub fn set_secret(key: String, value: String) -> Result<(), String> {
    Entry::new(SERVICE, &key)
        .map_err(|e| e.to_string())?
        .set_password(&value)
        .map_err(|e| e.to_string())
}

#[tauri::command]
pub fn get_secret(key: String) -> Result<Option<String>, String> {
    let entry = Entry::new(SERVICE, &key).map_err(|e| e.to_string())?;
    match entry.get_password() {
        Ok(value) => Ok(Some(value)),
        Err(Error::NoEntry) => Ok(None),
        Err(e) => Err(e.to_string()),
    }
}

#[tauri::command]
pub fn delete_secret(key: String) -> Result<(), String> {
    let entry = Entry::new(SERVICE, &key).map_err(|e| e.to_string())?;
    match entry.delete_credential() {
        // 이미 없는 항목을 지우는 것은 성공으로 친다 (멱등).
        Ok(()) | Err(Error::NoEntry) => Ok(()),
        Err(e) => Err(e.to_string()),
    }
}
```

> **이 모듈에는 Rust 유닛 테스트를 넣지 않는다.** 세 함수 모두 실제 OS 키체인을 건드리는 얇은 래퍼라 로직이 없고, CI(headless ubuntu)에는 Secret Service 데몬이 없어 `cargo test`가 반드시 실패한다. 검증 가치는 Task 2의 TS 마이그레이션 테스트와 Task 6의 수동 확인에 있다.

### Task 2: TS 시크릿 게이트웨이와 마이그레이션

시크릿에 접근하는 유일한 통로를 만든다. 이 모듈이 Tauri/브라우저 분기와 레거시 이관을 전부 흡수하므로, 이후 태스크의 호출부는 `getSecret`/`setSecret`만 알면 된다.

**읽기 경로의 레거시 폴백이 이 설계의 핵심이다.** 키체인에 항목이 없으면 localStorage를 읽어본다. 덕분에 마이그레이션이 아직 안 돌았거나 도는 중이어도 앱이 정상 동작하고, 부팅 시 마이그레이션(Task 5)과 화면 렌더 사이의 경쟁 조건이 사라진다. 마이그레이션이 평문을 지우고 나면 이 폴백은 자연히 `null`을 반환하므로 스스로 소멸한다.

**Files:**
- Create: `src/lib/secrets.ts`
- Create: `src/lib/secrets.test.ts`

**Interfaces:**
- Consumes: Task 1의 커맨드 3개
- Produces: 이후 모든 태스크가 쓴다.
  - `SECRET_KEYS: readonly ['github_pat', 'sync_webdav_pass', 'sync_encryption_key']`
  - `type SecretKey = (typeof SECRET_KEYS)[number]`
  - `getSecret(key: SecretKey): Promise<string | null>`
  - `setSecret(key: SecretKey, value: string): Promise<void>`
  - `deleteSecret(key: SecretKey): Promise<void>`
  - `migrateSecretsFromLocalStorage(): Promise<void>`

- [ ] **Step 1: 실패하는 테스트 작성**

`src/lib/secrets.test.ts` 생성. `invoke`를 인메모리 가짜 키체인으로 목킹해 실제 동작을 모델링한다. `vi.mock`은 호이스팅되므로 가짜 저장소는 `vi.hoisted`로 만들어야 참조할 수 있다.

```ts
import { describe, it, expect, beforeEach, vi } from 'vitest';

const { keychain, tauri } = vi.hoisted(() => ({
  keychain: new Map<string, string>(),
  tauri: { enabled: true },
}));

vi.mock('@tauri-apps/api/core', () => ({
  isTauri: () => tauri.enabled,
  invoke: vi.fn(async (cmd: string, args: { key: string; value?: string }) => {
    if (cmd === 'set_secret') { keychain.set(args.key, args.value!); return null; }
    if (cmd === 'get_secret') { return keychain.get(args.key) ?? null; }
    if (cmd === 'delete_secret') { keychain.delete(args.key); return null; }
    throw new Error(`unexpected command: ${cmd}`);
  }),
}));

import { getSecret, setSecret, deleteSecret, migrateSecretsFromLocalStorage } from '@/lib/secrets';

describe('secrets', () => {
  beforeEach(() => {
    keychain.clear();
    window.localStorage.clear();
    tauri.enabled = true;
  });

  describe('Tauri 경로', () => {
    it('키체인에 저장하고 다시 읽는다', async () => {
      await setSecret('github_pat', 'ghp_abc');

      expect(keychain.get('github_pat')).toBe('ghp_abc');
      expect(await getSecret('github_pat')).toBe('ghp_abc');
    });

    it('시크릿을 평문 localStorage에 남기지 않는다', async () => {
      await setSecret('github_pat', 'ghp_abc');

      expect(window.localStorage.getItem('github_pat')).toBeNull();
    });

    it('저장할 때 남아있던 레거시 평문을 치운다', async () => {
      window.localStorage.setItem('github_pat', 'ghp_legacy');

      await setSecret('github_pat', 'ghp_new');

      expect(window.localStorage.getItem('github_pat')).toBeNull();
      expect(await getSecret('github_pat')).toBe('ghp_new');
    });

    it('없는 시크릿은 null을 반환한다', async () => {
      expect(await getSecret('github_pat')).toBeNull();
    });

    it('키체인이 비었으면 레거시 평문을 읽는다 (마이그레이션 전에도 앱이 뜬다)', async () => {
      window.localStorage.setItem('github_pat', 'ghp_legacy');

      expect(await getSecret('github_pat')).toBe('ghp_legacy');
    });

    it('삭제하면 키체인과 레거시 평문이 모두 사라진다', async () => {
      window.localStorage.setItem('github_pat', 'ghp_legacy');
      await setSecret('github_pat', 'ghp_abc');

      await deleteSecret('github_pat');

      expect(await getSecret('github_pat')).toBeNull();
      expect(window.localStorage.getItem('github_pat')).toBeNull();
    });
  });

  describe('브라우저 경로', () => {
    beforeEach(() => { tauri.enabled = false; });

    it('localStorage를 쓰고 키체인을 건드리지 않는다', async () => {
      await setSecret('github_pat', 'ghp_abc');

      expect(window.localStorage.getItem('github_pat')).toBe('ghp_abc');
      expect(await getSecret('github_pat')).toBe('ghp_abc');
      expect(keychain.size).toBe(0);
    });
  });

  describe('마이그레이션', () => {
    it('시크릿 3종을 모두 키체인으로 옮기고 평문을 지운다', async () => {
      window.localStorage.setItem('github_pat', 'ghp_abc');
      window.localStorage.setItem('sync_webdav_pass', 'dav_pw');
      window.localStorage.setItem('sync_encryption_key', 'passphrase');

      await migrateSecretsFromLocalStorage();

      expect(keychain.get('github_pat')).toBe('ghp_abc');
      expect(keychain.get('sync_webdav_pass')).toBe('dav_pw');
      expect(keychain.get('sync_encryption_key')).toBe('passphrase');
      expect(window.localStorage.getItem('github_pat')).toBeNull();
      expect(window.localStorage.getItem('sync_webdav_pass')).toBeNull();
      expect(window.localStorage.getItem('sync_encryption_key')).toBeNull();
    });

    it('시크릿이 아닌 값은 건드리지 않는다', async () => {
      window.localStorage.setItem('sync_webdav_url', 'https://dav.example.com');
      window.localStorage.setItem('sync_webdav_user', 'alice');
      window.localStorage.setItem('data_retention_days', '90');

      await migrateSecretsFromLocalStorage();

      expect(window.localStorage.getItem('sync_webdav_url')).toBe('https://dav.example.com');
      expect(window.localStorage.getItem('sync_webdav_user')).toBe('alice');
      expect(window.localStorage.getItem('data_retention_days')).toBe('90');
      expect(keychain.size).toBe(0);
    });

    it('키체인 쓰기가 실패하면 평문을 지우지 않는다', async () => {
      // sync_encryption_key를 잃으면 원격 백업을 영구히 복호화할 수 없다.
      const { invoke } = await import('@tauri-apps/api/core');
      vi.mocked(invoke).mockRejectedValueOnce(new Error('keychain locked'));
      window.localStorage.setItem('sync_encryption_key', 'passphrase');

      await migrateSecretsFromLocalStorage();

      expect(window.localStorage.getItem('sync_encryption_key')).toBe('passphrase');
    });

    it('읽어서 대조한 값이 다르면 평문을 지우지 않는다', async () => {
      const { invoke } = await import('@tauri-apps/api/core');
      vi.mocked(invoke).mockImplementationOnce(async () => null)   // set_secret: 성공한 척
        .mockImplementationOnce(async () => 'corrupted');           // get_secret: 다른 값
      window.localStorage.setItem('sync_encryption_key', 'passphrase');

      await migrateSecretsFromLocalStorage();

      expect(window.localStorage.getItem('sync_encryption_key')).toBe('passphrase');
    });

    it('브라우저에서는 아무것도 하지 않는다', async () => {
      tauri.enabled = false;
      window.localStorage.setItem('github_pat', 'ghp_abc');

      await migrateSecretsFromLocalStorage();

      expect(window.localStorage.getItem('github_pat')).toBe('ghp_abc');
      expect(keychain.size).toBe(0);
    });
  });
});
```

```ts
import { isTauri, invoke } from '@tauri-apps/api/core';

/**
 * OS 키체인에 보관하는 시크릿. 이 목록에 없는 localStorage 항목
 * (sync_webdav_url, sync_provider 등)은 시크릿이 아니므로 건드리지 않는다.
 */
export const SECRET_KEYS = ['github_pat', 'sync_webdav_pass', 'sync_encryption_key'] as const;
export type SecretKey = (typeof SECRET_KEYS)[number];

export async function getSecret(key: SecretKey): Promise<string | null> {
  if (typeof window === 'undefined') return null;
  if (!isTauri()) return localStorage.getItem(key);

  const stored = await invoke<string | null>('get_secret', { key });
  if (stored !== null) return stored;

  // 키체인에 없다면 아직 이관되지 않은 평문일 수 있다.
  // 이 폴백 덕분에 마이그레이션 완료 여부와 무관하게 앱이 동작한다.
  // 마이그레이션이 평문을 지우면 여기서 자연히 null이 된다.
  return localStorage.getItem(key);
}

export async function setSecret(key: SecretKey, value: string): Promise<void> {
  if (typeof window === 'undefined') return;
  if (!isTauri()) {
    localStorage.setItem(key, value);
    return;
  }

  // 키체인 실패 시 localStorage로 폴백하지 않는다.
  // 조용히 평문으로 쓰면 이 기능의 목적이 사라진다.
  await invoke('set_secret', { key, value });
  localStorage.removeItem(key);
}

export async function deleteSecret(key: SecretKey): Promise<void> {
  if (typeof window === 'undefined') return;
  localStorage.removeItem(key);
  if (isTauri()) await invoke('delete_secret', { key });
}

/**
 * 0.4.3 이하에서 평문으로 저장된 시크릿을 키체인으로 옮긴다.
 * 키체인에 쓰고 읽어서 대조한 뒤에만 평문을 지운다 —
 * sync_encryption_key를 잃으면 원격 백업을 영구히 복호화할 수 없다.
 */
export async function migrateSecretsFromLocalStorage(): Promise<void> {
  if (typeof window === 'undefined' || !isTauri()) return;

  for (const key of SECRET_KEYS) {
    const legacy = localStorage.getItem(key);
    if (legacy === null) continue;

    try {
      await invoke('set_secret', { key, value: legacy });

      const verified = await invoke<string | null>('get_secret', { key });
      if (verified !== legacy) {
        console.error(`Secret migration verification failed for ${key}; keeping the plaintext copy.`);
        continue;
      }

      localStorage.removeItem(key);
    } catch (e) {
      console.error(`Secret migration failed for ${key}; keeping the plaintext copy.`, e);
    }
  }
}
```
